# XR-093. Авиарежим: раздел Files встречает ошибкой про не найденные шары, интерфейс подвисает

## Симптом

В авиарежиме вход в раздел файлов показывает ошибку, что шары не найдены,
хотя список шар уже известен и лежит в сторе. Несколько раз при этом
подвисал интерфейс приложения.

## Воспроизведение

1. Включить авиарежим на устройстве.
2. Открыть приложение и перейти в раздел Files.

Ожидаемо: сохранённый список шар открывается сразу (пусть и с пометкой,
что сети нет), интерфейс живой.

Фактически: ошибка «шары не найдены», интерфейс местами подвисает.

## Первые зацепки

- Вход в раздел зовёт `refreshHub` -> `repo.inviteShares` (JNI
  `nativeInviteShares` с таймаутом); при отказе сети наверх уходит
  `message = "Шары по инвайту: ..."`
  ([FilesViewModel.kt:100](../../xr-android/app/src/main/java/com/xrproxy/app/ui/files/FilesViewModel.kt#L100)).
  Ошибка обновления списка не должна перекрывать уже сохранённые шары.
- Подвисание UI требует отдельного разбора: сам сетевой вызов уже обёрнут
  в `Dispatchers.IO`, значит главный поток блокирует что-то ещё
  (кандидаты: другие JNI-вызовы из composable-пути, работа со стором).
- Смежно с XR-059: cache-first с пометкой «Офлайн» закрывает и этот
  сценарий, но подвисание интерфейса это отдельный дефект, его чинить в
  любом случае.

## DoD

- В авиарежиме раздел Files открывается на сохранённых данных без ошибки,
  недоступность сети видна ненавязчиво (в духе XR-059).
- Найдена и устранена причина подвисания интерфейса (блокировка главного
  потока), а не только замаскирован симптом.
