# XR-099: открытая шара не замечает вернувшуюся сеть, онлайн только после перезахода

Фидбек 2026-07-10: зайти в директорию шары без сети (показан локальный список
с пометкой «Офлайн»), затем включить сеть. Список так и остаётся офлайновым,
пока шару не переоткроешь. Выделено из XR-096 отдельной задачей.

## Корень

Cache-first (XR-059) стреляет `fetchManifest` один раз при `openShare`: упал
запрос, значит до `closeShare`/повторного `openShare` новых попыток нет.
Навигация по папкам внутри шары состояние не трогает
([FilesViewModel.kt](../../xr-android/app/src/main/java/com/xrproxy/app/ui/files/FilesViewModel.kt),
`navigateTo`), поэтому вернувшаяся сеть ничего не меняет.

## Что сделать

Перезапрашивать манифест при открытой шаре, когда есть повод думать, что сеть
вернулась. Кандидаты (выбрать при реализации):

- колбэк `ConnectivityManager` на появление validated-сети -> повторный fetch,
  если открыта шара с `offlineLocal`;
- и/или ретрай с бэкофом, пока шара открыта в `offlineLocal` (например 10с,
  30с, 60с, дальше по минуте);
- по приходу свежего манифеста молча подменить список и снять пометку (код
  подмены уже есть в `openShare`, вынести и переиспользовать).

## Проверка

1. В офлайне открыть шару со скачанными файлами, зайти в подпапку.
2. Включить сеть, шару не переоткрывать. Ожидаемо: в пределах десятков секунд
   список подменяется полным манифестом, пометка «Офлайн» уходит, текущая
   папка сохраняется.
