# XR-033: установка приложения по инвайту, страница /view как воронка

Получатель инвайта без приложения открывает `GET /invite/<token>/view` в
браузере. Раньше страница показывала только метаданные и QR с подписью «Scan
this QR code in the xr-proxy app to connect», то есть предполагала, что
приложение уже стоит. Для того, кому ссылку прислали в первый раз, это тупик.

LLD-04 (п. 5.2) закладывал такую воронку. Страница пытается открыть приложение
через deep link, а если его нет, ведёт на скачивание APK.

## Что сделано

- `xr-proto::invite_url::build_custom_url(hub_url, token)` строит
  гарантированный deep link `xr://invite/<token>?hub=<host>` (симметрично с
  `build_https_url`). На странице /view приложение заведомо не дефолтный
  обработчик ссылки (иначе получатель не смотрел бы страницу в браузере),
  поэтому кастомная схема надёжнее HTTPS: её перехватывает установленный
  клиент напрямую, без chooser'а, и она не уводит обратно в браузер.
- Страница `/view` (`view_invite` в `xr-hub`) переделана в воронку:
  - кнопка «Открыть в приложении» с deep link (гасится, когда инвайт не
    active, чтобы не вести в claim, который вернёт 410);
  - кнопка «Скачать APK» на `/api/v1/app/download/latest` (раздача релиза
    из LLD-12, латест-алиас всегда тянет свежую сборку);
  - подсказка «нет приложения? скачайте APK и вернитесь по ссылке»;
  - QR остаётся для сканирования встроенным сканером с другого устройства.
  - тексты переведены на русский (страницу видит русскоязычный получатель).

`assetlinks.json` (опциональный пункт задачи) не заводим: манифест Android
намеренно без `autoVerify` (домены self-hosted хабов у каждого свои, LLD-04
п. 3.4), поэтому App Links verification неприменима и файл ничего не даст.
Гарантированный переход в приложение обеспечивает кастомная схема.

## Проверка

Юнит-тесты:

- `xr-proto`: `build_custom_roundtrips_with_parse`,
  `build_custom_keeps_port_strips_scheme_and_slash`.
- `xr-hub`: `view_offers_deep_link_and_apk` (на странице есть deep link и
  кнопка APK), `view_disables_open_for_expired_invite` (у просроченного
  инвайта кнопка открытия погашена).

Руками: открыть `/api/v1/invite/<token>/view` в мобильном браузере, на
устройстве с приложением тап «Открыть в приложении» ведёт в экран
подтверждения инвайта; на чистом устройстве «Скачать APK» отдаёт свежий
релиз.
