# XR-059: офлайн-просмотр Files по факту не работает

Фолбэк XR-053 на месте, но без сети в разделе крутится индикатор, выходит ошибка
загрузки шар, локальный список файлов так и не появляется (фидбек 2026-07-03).

## Корень

Фолбэк включается только после того, как сетевой запрос честно упадёт, а падает
он долго. `openShare` сначала ждёт `fetchManifest` (общий таймаут 60с) и лишь по
его ошибке строит локальный манифест; при поднятом VPN с недоступным сервером
коннект принимает локальный smoltcp-стек (SYN-ACK до апстрима,
[engine.rs:474](../../../../xr-core/src/engine.rs#L474)), поэтому connect-timeout 10с
не спасает и запрос может висеть до полного таймаута. Плюс на входе во вкладку
всегда стреляет `refreshHub` (таймаут 15с): своя крутилка + тост «Шары по
инвайту: ...». Пользователь видит крутилку с ошибкой и уходит раньше, чем фолбэк
дожил бы до показа (XR-053 проверялся на мгновенном отказе сети, авиарежим без
VPN, там таймауты не проявлялись).

## Что сделать: cache-first

- При открытии шары сразу показывать локальный манифест с пометкой «Офлайн»,
  свежий тянуть фоном и подменять по приходу.
- При показанном локальном списке сетевую ошибку не тостить.
- `refreshHub` при офлайне не ронять в тост поверх рабочего локального контента.
- Если локально ничего не скачано, показывать честное «нет сети и нет
  скачанного», а не голую ошибку.

## Что сделано

Пункт про `refreshHub` закрыла смежная XR-093 (флаг `hubOffline` со строкой
«Хаб недоступен...» вместо тоста). Остальное здесь,
[FilesViewModel.kt](../../../../xr-android/app/src/main/java/com/xrproxy/app/ui/files/FilesViewModel.kt),
[FilesScreen.kt](../../../../xr-android/app/src/main/java/com/xrproxy/app/ui/files/FilesScreen.kt):

- `openShare` перевёрнут на cache-first: сначала быстрый локальный манифест
  (скан файловой системы) и сразу на экран с пометкой «Офлайн: показаны только
  скачанные файлы», затем сетевой `fetchManifest` фоном; пришедший свежий
  список молча подменяет локальный, пометка снимается.
- Сетевая ошибка фонового запроса (`network:` из native-слоя) при показанном
  локальном списке не тостится вообще: пометка «Офлайн» уже всё говорит.
  Не-сетевые ответы агента (протухший токен, `http_4xx`) остаются тостом,
  как в XR-093.
- Офлайн при пустом локальном кэше показывает в теле списка «Сети нет, а
  скачанных файлов пока нет» (текст согласован) вместо тоста с голой ошибкой;
  пометка «Офлайн...» при пустом списке не рисуется, чтобы не противоречить
  самой себе.
- Крутилка осталась только на случай «кэша нет, сеть отвечает»: там честно
  ждём манифест.

## Проверка

1. Поднять VPN при недоступном сервере (или Wi-Fi без интернета) -> Files ->
   открыть шару со скачанными файлами. Ожидаемо: локальный список виден сразу
   (1-2 с) с пометкой «Офлайн: показаны только скачанные файлы», без минутной
   крутилки и без тостов; скачанные файлы открываются.
2. Восстановить сеть, переоткрыть шару. Ожидаемо: список подменяется полным
   манифестом, пометка «Офлайн» уходит (при живой сети она мелькает на время
   загрузки манифеста, это ожидаемо).
3. В офлайне открыть шару, из которой ничего не скачано. Ожидаемо: вместо
   крутилки и тоста «Манифест: network: ...» в теле списка «Сети нет, а
   скачанных файлов пока нет».
