# XR-052: импорт контента по URL как плагин (поверх записи XR-139)

Дизайн в [LLD-29](../lld/29-share-url-import.md); тут постановка и ход задачи.
Тип задачи LLD: результат это сам дизайн, реализация пойдёт отдельной задачей
после согласования (как XR-051 -> XR-139).

## Постановка

- **UI**: в writable-шаре зайти в папку -> «Импорт по URL» -> вставить ссылку
  на страницу -> агент качает контент в эту папку с адекватным именем из
  метаданных (например заголовок ролика).
- **Архитектура**: не прибито к xr: общий интерфейс плагина-фетчера,
  референс-плагин это обёртка yt-dlp + ffmpeg, ядро xr-share остаётся тонким
  файлсервером; роутинг URL -> плагин по паттерну, плагины опциональны
  (владелец включает).
- **Безопасность**: агент в чужой квартире, URL от пользователя: dest строго
  в пределах шары, url литеральным аргументом без shell, SSRF-защита, лимиты
  размер/диск/таймаут, только держателям права записи, по возможности
  песочница плагина.

## Проверка (сценарий для Check)

Задача дизайновая, проверка это согласование LLD-29:

1. Прочитать [LLD-29](../lld/29-share-url-import.md), в первую очередь
   решения п. 3 и снятые развилки п. 8 (скоуп `share:import` вместе с write,
   процесс на джобу без демона, суффиксы хоста вместо regex, поллинг джобы,
   резерв неймспейса `.xr-`).
2. Отдельно взглянуть на тексты Android UI (п. 2.8): действие «Импорт по
   URL», диалог «Ссылка» / «Импортировать», строка «Импорт: N%».
3. Согласен -> закрыть задачу и завести задачу реализации LLD-29; замечания
   -> вернуть в In progress, правки поедут в сам LLD.
