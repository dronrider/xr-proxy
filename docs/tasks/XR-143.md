# XR-143: набор доступных сервисов на инвайте

Владелец управляет тем, какие сервисы хаба открывает инвайт: только VPN, VPN
плюс шары, только шары; дальше в набор встанет мессенджер (XR-076). Получатель
видит в приложении только доступное, лишний функционал скрыт.

## Текущее состояние

- VPN зашит в инвайт безусловно: `InvitePayload` всегда несёт серверы и ключ
  обфускации ([invites.rs](../../xr-hub/src/api/invites.rs), `build_invite`),
  инвайт «только шары» невозможен.
- Шары привязываются точечно (`share_ids` / `write_share_ids`, чтение и запись
  из LLD-28), но это наполнение, признака «сервис Files включён» нет.
- Приложение ничего не скрывает: вкладки VPN / Log / Servers / Files статичны
  ([MainActivity.kt](../../xr-android/app/src/main/java/com/xrproxy/app/ui/MainActivity.kt)).

## Решено с владельцем 2026-07-19

- Набор сервисов задаётся явным полем на инвайте (VPN, Files, дальше
  мессенджер), а не выводится из наполнения: случай «Files включён, шары
  добавлю позже» и сервисы без наполнения иначе не выразить.
- Доступ следует за инвайтом. Владелец меняет набор у уже принятого инвайта,
  хаб проверяет текущий набор при каждом запросе, приложение подхватывает
  изменения при очередном обращении к хабу и перестраивает навигацию.
  Расширение и сужение доступа без переонбординга; это же закрывает открытый
  вопрос из раздела 6 LLD-25 (scope следует за инвайтом).
- Оформление отдельным LLD, не расширением LLD-25: тот остаётся про жизненный
  цикл инвайта, здесь модель набора сервисов и скрытие в UI.

## Рамка для LLD

- Набор сервисов ложится в `scope` мандата XR-122; пока мандата нет, проверка
  идёт по инвайту.
- До per-invite идентичности (XR-074) запрет VPN держится на невыдаче конфига:
  серверы и ключ в payload выключенного VPN не кладутся. Кто ключ уже получил,
  того так не отрезать; честный запрет на туннеле уедет в XR-074, для
  тестового парка невыдачи достаточно.
- Скрытие в приложении опирается на модель XR-058 (сервисы как навигация
  внутри хаба-аккаунта): набор вкладок определяет активный хаб. Отдельный
  вопрос LLD: главный экран при инвайте без VPN, сейчас приложение
  VPN-центрично.
- Наполнение внутри сервиса остаётся своими задачами: какие шары (XR-102),
  какой пресет (XR-119).
- Набор сервисов это по сути права доступа, на них лягут лимиты и тарифы
  монетизации (XR-073).

## Смежные задачи

XR-122 (мандат со scope), XR-058 (навигация по сервисам), XR-030 (инвайт ->
JWT, сервисы в claims), XR-036 (скоупы consume/publish у инвайтов агентов),
XR-074 (запрет на туннеле), XR-076 (мессенджер как сервис), XR-073
(монетизация), XR-102/XR-119 (наполнение внутри сервиса).
