# XR-100: на мобильном интернете шара офлайн при живом внешнем пути

Фидбек 2026-07-10: в домашней сети (Wi-Fi, там же сервер шары) вход в шару
показывает файлы онлайн; после переключения на мобильный интернет вход в ту же
шару показывает «Офлайн», хотя раздача с внешнего адреса работает.

## Диагностика на 2026-07-10

Внешний путь до агента проверен и жив: с обеих VPS (роль exit для туннеля
телефона) `GET /manifest` внешнего адреса агента отвечает 401 (нет токена, но
TCP/HTTP живой) за ~0.1с. Проброс на роутере цел, hairpin для локальной сети
тоже (пункт 1 фидбека работает как раз через него, VPN в домашней сети у
телефона в паузе по доверенному SSID).

На LTE VPN активен, HTTP приложения идёт через TUN в движок (приложение себя
из VPN не исключает), дальше либо Direct с телефона, либо relay через VPS. Оба
плеча снаружи работают, поэтому корень надо ловить на телефоне: нужна строка
«манифест шары ...: network: ...» из вкладки «Журнал» при воспроизведении на
LTE (детали ошибки: timeout / refused / no route) и состояние VPN в этот
момент (активен, пауза, момент переключения сетей).

Возможная связь с XR-099: если репро было сразу после переключения Wi-Fi ->
LTE, запрос мог попасть в пересборку туннеля при смене сети (C4 recycle), а
дальше без перезахода в шару офлайн уже не снимается. Тогда XR-100 сведётся к
дублю XR-099.

## Проверка

На мобильном интернете (спустя минуту после переключения сетей, VPN активен)
вход в шару показывает полный список без пометки «Офлайн» за секунды.
