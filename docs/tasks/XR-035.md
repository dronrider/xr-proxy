# XR-035: доступ к шаре без белого IP (агент за NAT/CGNAT)

Недостижимый снаружи агент остаётся доступен потребителю через посредника.
Спроектировать архитектуру.

## Развилка

Реверс-туннель агент -> хаб/relay (все байты идут через посредник, дорого по
egress на шарах в десятки ГБ) против hole-punching агент <-> потребитель с relay
как fallback (байты напрямую, экономит трафик, но CGNAT/симметричный NAT часто
требует fallback).

## Учесть

- E2E (relay не видит плейнтекст);
- хаб как слой управления (сигналинг), а передача байт отдельным relay-сервисом;
- переиспользование обфусцированного mux из xr-core/xr-server как транспорта;
- целостность байт через подписанный манифест (XR-046).

При проектировании LLD возможна разбивка на relay (работает всегда) и
hole-punching (оптимизация трафика).
