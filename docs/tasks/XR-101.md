# XR-101: «Мои шары» не замечают удаление шары на хабе

Фидбек 2026-07-10: владелец удалил шару (`xr-share unshare`), а у коллеги она
осталась в списке, выглядит как «закэшировалась намертво».

## Корень

Добавленная шара персистится в локальный стор телефона и живёт там до ручного
удаления. Обновление по инвайту наполняет только секцию «Доступно по инвайту»
и обновляет реквизиты уже известных шар (`reconcileShares`), но пропажу шары
из грантов никак не обрабатывает: карточка в «Мои шары» остаётся без какого
либо признака, что источника больше нет. Это отчасти задумано (скачанные
файлы должны жить и без хаба), но сигнала «источник удалён» нет вовсе, и
пользователь читает это как вечный кэш. Хаб при этом чистый: удалённая шара
из выдачи `/invite/{token}/shares` пропадает сразу (реестр живой, протухшие
привязки фильтруются).

## Что сделать

- При успешном обновлении по инвайту сравнивать сохранённые шары с грантами:
  пропавшую помечать в карточке («источник удалён владельцем» или подобное,
  текст согласовать) и предлагать убрать из списка; скачанные файлы при
  удалении карточки не трогать (как сейчас).
- Не путать с временной недоступностью хаба: помечать только по успешному
  ответу хаба, в котором шары нет (hubOffline неповод).
- Учесть шары, добавленные прямой ссылкой (`xrshare://`, без инвайта): их в
  грантах нет и не будет, помечать нельзя. Нужен признак происхождения шары
  в ShareConfig.

## Проверка

1. Добавить шару по инвайту на два устройства, на одном удалить её агентом
   (`xr-share unshare`), на втором обновить список. Ожидаемо: карточка
   помечена как удалённая источником, по явному действию убирается из списка,
   скачанные файлы на диске целы.
2. Выключить хаб и обновить список: сохранённые шары без пометок (офлайн
   хаба не считается удалением).
