# XR-177: Автоустановка (ZTP), этап 1б: router-профиль `xr-setup` (procd, nftables, dnsmasq на Quad9, отложенный SSID, enroll в реестр LLD-17). Поверх каркаса XR-015

Дизайн в [LLD-13](../lld/13-zero-touch-provisioning.md) (разделы 2.1, 3.2, 5.6,
5.9), отличия реализации перенесены туда же в раздел 8. Каркас шагов и
server-профиль сделаны в XR-015 (архив 2026).

## Рамки

- Профиль `router` в `xr-setup`: доставка `xr-client` из setup-dist, рендер
  `config.toml` в формате пула `[[servers]]` (LLD-10), procd init c fd-limit,
  watchdog и nftables-скрипты, sysctl буферов, dnsmasq на Quad9, отложенная
  смена SSID последним шагом (LLD-13 п. 5.9).
- Enroll в реестр хаба: клиентская сторона шва с LLD-17 (`POST /api/v1/enroll`
  по `--enroll-token`, запись `[control]` в конфиг). Серверная сторона реестра
  это XR-025, до неё шаг с токеном честно падает ошибкой хаба; без токена
  роутер автономен, как и описано в LLD.
- В `deploy/xr-proxy.init` закрыт дрейф с живым флотом: на роутерах в init
  стоит `procd_set_param limits nofile="16384 16384"`, в репозитории строки
  не было.

## Ход работы

- Роутер DE сверен как эталон раскладки (read-only): fd-limit через procd
  limits, dnsmasq с `noresolv=1` и двумя апстримами Quad9, wifi-iface секции
  обоих радио.
- Секцию `[control]` клиент сегодня игнорирует (serde без deny_unknown_fields),
  запись шва безопасна до реализации XR-025.
- Перезапуск сервисов в шагах обобщён (`Restart::Unit`/`Restart::Initd`),
  скрипты, в отличие от конфигов, всегда приводятся к вшитой версии.
- SSID меняется только у ap-секций wireless, sta-аплинк репитера не трогается;
  `wifi reload` уходит в фон с задержкой 10 секунд.
- Смоук в OpenWRT-контейнере (см. сценарий): чистая установка 10 шагов,
  повторный прогон полностью «уже настроено», verify зелёный. Docker Hub
  образ `openwrt/rootfs` не тянулся, rootfs берётся тарболлом с
  downloads.openwrt.org.
- SSID и enroll в контейнере не проверить (нет `/etc/config/wireless` и нет
  реестра до XR-025): оба покрыты юнитами (`ap_sections`, рендер и детекция
  `[control]`, разбор ответа enroll); живая проверка SSID поедет вместе с
  XR-171 (этап 2) либо первой реальной установкой роутера.

## Сценарий проверки (агентский)

Прод не трогается: установка гоняется в одноразовом aarch64-контейнере с
procd. Все шаги с машины агента, итоги объективные (код возврата и строки
вывода).

1. Собрать бинари и дист:
   `cargo zigbuild --release --target aarch64-unknown-linux-musl -p xr-setup -p xr-client`,
   сложить в `$DIST` как `xr-setup-linux-aarch64` и `xr-client-linux-aarch64`,
   рядом `SHA256SUMS` (`shasum -a 256 ... > SHA256SUMS`).
2. Поднять OpenWRT-контейнер (Docker Hub может не отдаваться, брать тарболл):
   `curl -fsSLO https://downloads.openwrt.org/releases/23.05.5/targets/armsr/armv8/openwrt-23.05.5-armsr-armv8-rootfs.tar.gz`,
   `docker import --platform linux/arm64 <тарболл> openwrt-rootfs:xr177`,
   `docker run -d --name xr177 --platform linux/arm64 --privileged -v $DIST:/dist openwrt-rootfs:xr177 /sbin/init`.
3. Вернуть контейнеру egress: netifd на старте сносит докерную сеть, поэтому
   внутри контейнера повесить на `br-lan` тот адрес, что докер выдал
   контейнеру (`docker network inspect bridge`), и дефолт-маршрут на шлюз
   моста (`ip addr add <addr>/16 dev br-lan; ip route add default via <gw>`).
4. Чистая установка:
   `docker exec xr177 /dist/xr-setup-linux-aarch64 router --server 203.0.113.1:8443 --key QUJDREVGRw== --salt 0xDEADBEEF --from-dir /dist`.
   Ожидание: exit 0, десять строк `[+]` (binary, config, пять скриптов,
   sysctl, service, dnsmasq), затем `[v]` процесс, `[v]` nftables `ip
   xr_proxy`, `[v]` DNS.
5. Идемпотентный повтор той же командой: exit 0, все шаги `уже настроено`,
   `применено: 0`, verify снова зелёный.
6. Артефакты на месте: `docker exec xr177 sh -c 'ls /etc/rc.d | grep S99xr-proxy && crontab -l | grep xr-watchdog && nft list tables | grep xr_proxy && grep nofile /etc/init.d/xr-proxy'` без ошибок.
7. `cargo test --workspace` зелёный, 0 warnings (на маке
   `--exclude xr-client`).
8. После выката раздачи: `curl -fsSL https://<хаб>/api/v1/setup/SHA256SUMS`
   содержит `xr-client-linux-aarch64` и `xr-client-linux-x86_64`.

Прогон 2026-07-24 (шаги 1-7) пройден, вывод шага 4/5 совпал с ожиданием
дословно; шаг 8 после выката setup-dist.
