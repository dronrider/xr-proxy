# XR-042: персистентный журнал логов вместо ленты из транзиентных источников

Логи не должны зависеть от состояния (сеть/режим), это просто журнал записей.

## Проблема

Сейчас лента собирается из транзиентных источников (engine `recentErrors` +
`probeLog`): при смене сети движок перезапускается и recentErrors обнуляется,
после переключения доверенная -> обычная тянутся пробы доверенной.

## Что сделать

Единый персистентный append-only журнал, переживающий перезапуск движка и смену
сети/режима (не обнуляется): все источники пишут в один буфер. Само переключение
сети/режима логировать отдельной записью. Ротацию (размер/кол-во) вынести в
настройки. Лог должен включать и файловый трафик/события (скачивания, синки,
ошибки шар), не только VPN (фидбек).
