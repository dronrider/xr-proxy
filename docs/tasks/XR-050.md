# XR-050: шара с несколькими адресами (LAN + публичный)

Потребитель выбирает достижимый адрес.

## Проблема

Сейчас у шары один addr: на одной LAN с агентом публичный IP недоступен из-за
NAT hairpin (роутер не заворачивает LAN -> свой WAN-IP обратно на 192.168.x), а
LAN-адрес агента недоступен снаружи.

## Что сделать

Хранить в записи шары список адресов (LAN-адреса агента + публичный/DDNS),
потребитель пробует их по очереди и берёт первый ответивший: внутри LAN по
LAN-IP без hairpin, снаружи по публичному. Роутеро-независимо, в отличие от
включения NAT reflection на роутере.

Код: [xr-share/src/cli.rs](../../xr-share/src/cli.rs),
[xr-core/src/sync.rs](../../xr-core/src/sync.rs)
