# XR-135: синк не удаляет скачанное при пустом выборе

Воспроизведение: отметить файлы, синкнуть, затем снять все галки и нажать синк.
Вместо удаления локальных копий тост «Отметь галочками файлы или папки для
синка». Зеркало обещает «удаляет лишнее», но снятое подчистится только пока
остаётся хотя бы одна галка; снять последнюю и удалить остаток синком нельзя.
Файлы висят на диске, пока пользователь не снесёт их руками файловым
менеджером (удаление шары локальные копии тоже не трогает).

## Корень

Пустой выбор трактуется тремя слоями по-разному, и все три вместе запирают
удаление:

- [FilesViewModel.syncNow](../../xr-android/app/src/main/java/com/xrproxy/app/ui/files/FilesViewModel.kt#L331)
  при пустом `selection` отказывает с тостом;
- [ShareRepository.syncOnce](../../xr-android/app/src/main/java/com/xrproxy/app/data/ShareRepository.kt#L151)
  при пустом выборе возвращает no-op (защита от того, чтобы фоновый воркер не
  утащил всю шару и не держал transfer lock); докстринг при этом до сих пор
  утверждает «An empty selection mirrors the whole share», код ниже делает
  обратное;
- JNI [nativeSyncShare](../../xr-android-jni/src/lib.rs#L1388) маппит `"[]"` в
  `None`, а `None` для движка значит «вся шара». Выразить «выбор пуст, снеси
  всё локальное» через JNI сейчас нельзя, хотя сам движок это умеет:
  `plan_with_selection` с `Some(пустое множество)` даёт план из одних delete.

Обе котлин-заглушки поставлены именно потому, что пустой массив на нижнем
слое означал бы полное зеркало шары. Семантика «пусто = вся шара» осталась и в
подписи карточки шары («вся шара» при пустом выборе в
[FilesScreen](../../xr-android/app/src/main/java/com/xrproxy/app/ui/files/FilesScreen.kt#L289)),
хотя фактически пустой выбор давно означает «ничего не синкается».

## Направление починки

В JNI развести null/отсутствие («вся шара») и `"[]"` (`Some(пустой набор)` ->
только удаление), после чего ручной `syncNow` с пустым выбором вместо тоста
делает delete-only проход. Удаление всего скачанного это разрушительное
действие, перед ним нужен диалог подтверждения («Галок нет: удалить всё
скачанное этой шары?»). Фоновый воркер оставить как есть (пустой выбор ->
no-op): авто-удаление всего по расписанию без явного действия пользователя
пугает сильнее, чем лишние файлы. Заодно поправить подпись «вся шара» и
протухший докстринг `syncOnce`.

Развилка: XR-044 (редизайн синк-UX) заменяет галочно-зеркальную модель на
per-file контрол с явным «минусом» на скачанном, и там проблема исчезает по
построению. Если XR-044 близко, чинить только доку/подпись и подтверждение не
городить; если далеко, delete-only проход дешёвый и снимает боль сейчас.

## Проверка

1. Отметить пару файлов, синкнуть -> файлы скачались.
2. Снять одну галку, синкнуть -> снятый файл удалился (это работает и сейчас).
3. Снять последнюю галку, синкнуть -> диалог подтверждения, после «да» все
   локальные копии шары удалены, тост с «-N», счётчик у иконки синка погас.
4. Отказ в диалоге -> файлы на месте.
5. Фоновый синк с пустым выбором по-прежнему ничего не удаляет и не качает.
