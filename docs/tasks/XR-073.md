# XR-073: монетизация через платный шаринг хаба

Модель владельца: владелец хаба ставит на инвайт лимиты (число добавленных
ресурсов серверы/шары; объём трафика; опц. устройства), а чтобы превысить,
пользователь покупает абонемент.

## Механика

Хаб ведёт учёт по инвайту и применяет лимит (мягкий/жёсткий отруб при
превышении, grace, продление). Лимит на ресурсы хаб считает у себя (он их и
выдаёт под инвайт); лимит на трафик требует приватного метринга (XR-075) и
per-invite идентичности на туннеле (XR-074, сейчас ключ один на всех,
атрибутировать трафик нечем). Абонемент это entitlement, поднимающий лимит;
приём платежей отдельный спайк по рельсам под РФ (СБП/ЮMoney, крипто/USDT,
Telegram Stars). Тарифы можно повесить и на скорость (профили XR-066), число
серверов, объём Files, egress relay.

## Приоритизация

Ценность низкая: владелец доход сейчас не приоритизирует, потому низший ранг.
Зависит от XR-074 и XR-075, смежно с XR-030, XR-035.
