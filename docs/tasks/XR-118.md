# XR-118: правила не подхватываются при resume из паузы и «включить здесь»

Найдено владельцем при проверке XR-047: добавил `*.2ip.ru -> proxy`, а ресурс
продолжил показывать реальный IP и с включённой доверенной сетью, и с
выключенной.

## Корень

Конфиг движка (`buildConfigJson`, включая массив `user_rules`) собирается
только в `VpnViewModel.actuallyStart`, то есть при полном Connect с главного
экрана. Сервис кэширует его в `XrVpnService.lastConfigJson` и все остальные
пути поднятия туннеля переиспользуют кэш как есть:

- resume из паузы при уходе из доверенной сети (`requestResume`);
- «включить здесь» с главной и из уведомления (`resumeOverride`, ветки
  Paused и Idle/Error);
- перезапуск после ошибки.

Правило, добавленное после последнего полного Connect, в эти пути не попадает.
Снекбар обещает «применятся при следующем подключении», но с точки зрения
пользователя resume это и есть следующее подключение. Дополнительно к сути
XR-047 не относится: в самой паузе VPN выключен целиком и весь трафик идёт
напрямую, правила в этом состоянии не действуют by design.

## Что делать

Перечитывать `user_rules.json` при каждом поднятии туннеля: в
`bringTunnelUp` (или на входе `startVpn`) подменять поле `user_rules` в
`lastConfigJson` свежим содержимым файла (org.json: распарсить, put, обратно
в строку). Сервис знает `filesDir`, VM для этого не нужна. Заодно решить,
надо ли так же освежать `default_action`/hub-поля (сейчас они меняются только
вместе с профилем, который без полного реконнекта не переключить, так что
достаточно user_rules).

## Шаги воспроизведения (до фикса)

1. Подключить VPN с главного экрана, дождаться Connected.
2. Уйти в доверенную сеть (авто-пауза) либо получить паузу любым способом.
3. Добавить правило `*.2ip.ru -> proxy` на экране правил.
4. Нажать «Включить здесь» (или выйти из доверенной сети, дождаться resume).
5. Открыть 2ip.ru: показывает реальный IP (правило не действует), хотя
   туннель поднят. Ожидаемо после фикса: IP прокси-сервера.

## Сценарий проверки (после фикса)

Шаги те же; на шаге 5 exit-IP для 2ip.ru равен адресу VPS (проверять по
IP на странице, а не по факту её открытия). Контроль: полный Disconnect +
Connect ведёт себя как раньше.
