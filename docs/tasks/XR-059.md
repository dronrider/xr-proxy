# XR-059: офлайн-просмотр Files по факту не работает

Фолбэк XR-053 на месте, но без сети в разделе крутится индикатор, выходит ошибка
загрузки шар, локальный список файлов так и не появляется (фидбек 2026-07-03).

## Корень

Фолбэк включается только после того, как сетевой запрос честно упадёт, а падает
он долго. `openShare` сначала ждёт `fetchManifest` (общий таймаут 60с) и лишь по
его ошибке строит локальный манифест; при поднятом VPN с недоступным сервером
коннект принимает локальный smoltcp-стек (SYN-ACK до апстрима,
[engine.rs:474](../../xr-core/src/engine.rs#L474)), поэтому connect-timeout 10с
не спасает и запрос может висеть до полного таймаута. Плюс на входе во вкладку
всегда стреляет `refreshHub` (таймаут 15с): своя крутилка + тост «Шары по
инвайту: ...». Пользователь видит крутилку с ошибкой и уходит раньше, чем фолбэк
дожил бы до показа (XR-053 проверялся на мгновенном отказе сети, авиарежим без
VPN, там таймауты не проявлялись).

## Что сделать: cache-first

- При открытии шары сразу показывать локальный манифест с пометкой «Офлайн»,
  свежий тянуть фоном и подменять по приходу.
- При показанном локальном списке сетевую ошибку не тостить.
- `refreshHub` при офлайне не ронять в тост поверх рабочего локального контента.
- Если локально ничего не скачано, показывать честное «нет сети и нет
  скачанного», а не голую ошибку.

## Проверка

Шаги: поднять VPN при недоступном сервере (или Wi-Fi без интернета) -> Files ->
открыть шару со скачанными файлами. Ожидаемо: локальный список виден сразу
(1-2 с) с пометкой «Офлайн» без минутной крутилки и тостов, после восстановления
сети список подменяется полным манифестом.
