# XR-057: камуфляж под настоящий TLS (потолок неотличимости)

Класс Reality/VLESS+TLS. Криптоядро (forward secrecy вместо XOR) вынесено в
XR-061, анти-фингерпринт первого пакета (убрать magic, junk-пакеты) в XR-062,
здесь остаётся обёртка потока в честный TLS.

## Проблема

Сейчас транспорт это чистый XOR-поток
([obfuscation.rs](../../xr-proto/src/obfuscation.rs),
[protocol.rs](../../xr-proto/src/protocol.rs)): со второго байта высокая
энтропия без структуры, первый пакет не похож ни на TLS, ни на HTTP, это класс
fully-encrypted, который DPI ловит по энтропии и активному зондированию.
Реактивный HTTP-fallback ([xr-server/src/fallback.rs](../../xr-server/src/fallback.rs))
от активного зондирования не спасает (отвечает только после 4096 байт мусора).

## Развилка (спайк)

- (б) обёртка в настоящий TLS/WS к своему домену (класс VLESS+TLS): нужен
  домен+серт, SNI палится и блокируется по нему;
- (в) Reality-класс (угон TLS-хендшейка чужого сайта без своего домена,
  форвардинг активных проб на реальный бэкенд): актуальный потолок, но своя
  реализация в Rust это крупный и рисковый протокол;
- (г) переиспользовать обкатанный внешний слой (sing-box/Xray, ss-2022) под наш
  mux.

## Приоритизация

Серьёзность 0 пока обход работает, при ужесточении ТСПУ это Blocker без
пользовательского обхода (ранг поднимется сам). Лист протокола v2, зависит от
XR-061 (XR-060). Смежно с XR-062, XR-009, XR-010.
