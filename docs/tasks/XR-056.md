# XR-056: компактный общий индикатор очереди синка

Общего индикатора прогресса синка по сути нет, а место он занимает много.

## Проблема

Текущий
[ProgressBar](../../xr-android/app/src/main/java/com/xrproxy/app/ui/files/FilesScreen.kt#L468)
это Card во всю ширину про один файл: имя + полоса по байтам текущего файла +
скорость, а агрегат «файл N/M» втиснут мелким текстом.

## Что сделать

Компактный общий индикатор очереди: какой файл синкается сейчас, сколько
осталось (X из N), общий прогресс по батчу, стоп всей очереди. Компактно (одна
строка / тонкая полоса), не карточка на пол-экрана. Данные уже приходят в
`Progress` (`filesDone`/`filesTotal`/`file`, движок отдаёт
`files_done`/`files_total`), нужно агрегатное представление вместо по-файлового.
Детали конкретного файла (байты/скорость) из общего индикатора убрать, они
переезжают на строку файла (XR-044). Виден и для фонового синка, не только
foreground.

Смежно с XR-044 (та же панель/экран, прогресс на строках это предпосылка ужать
глобальный), логично делать в один заход.
