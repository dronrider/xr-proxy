# XR-102: привязать шару к инвайту можно только в момент создания

Фидбек 2026-07-10: владелец добавил вторую шару, и она не появилась на его
телефоне, хотя коллега её видела. Выглядело как «список закэшировался».

## Корень

Шара видна в приложении только через инвайт, к которому привязана
(`/invite/{token}/shares`). Привязка делается единственным способом: флагом
`--invite` (повторяемым) в момент `xr-share share`. В итоге:

- забыл привязать к своему инвайту, и сам не видишь свою шару (ровно случай
  фидбека: на хабе у инвайта владельца привязок ноль, у инвайта коллеги обе);
- привязать существующую шару к ещё одному инвайту из CLI нельзя вообще,
  эндпоинты хаба `/share/attach`, `/share/detach` есть, а команды нет;
- `xr-share list` не показывает, к каким инвайтам шара привязана, диагностика
  «почему не видно» вслепую.

Смежное: `unshare` удаляет шару из реестра хаба, но её `share_id` остаётся
висеть в `share_ids` инвайтов навсегда (выдача это фильтрует, но мусор
копится); чистить привязки при удалении шары.

## Что сделать

- Команды `xr-share attach <share> <invite>` и `detach` поверх существующих
  эндпоинтов хаба.
- В `xr-share list` показывать привязки шары (какие инвайты).
- В выводе `xr-share share` без `--invite` предупреждать явно, что в
  приложениях шару не увидит никто, включая владельца.
- На хабе при удалении шары вычищать её из `share_ids` всех инвайтов.

## Проверка

1. `xr-share attach` существующей шары к инвайту телефона: после обновления
   списка шара появляется в «Доступно по инвайту».
2. `xr-share list` показывает привязки; `detach` убирает шару из выдачи
   этого инвайта.
3. После `unshare` в json-файлах инвайтов на хабе не остаётся её share_id.
