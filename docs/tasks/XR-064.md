# XR-064: UDP/QUIC-транспорт как альтернатива одиночному TCP

Для мобильных и потерянных линков (класс Hysteria2/AmneziaWG/TUIC).

## Проблема

Сейчас туннель ходит по TCP + mux-пул
([mux_pool.rs](../../xr-proto/src/mux_pool.rs)): пул снимает межстримовый HoL,
но не саму чувствительность TCP к потерям, а loss-based CC проседает на LTE.

## Что спроектировать

QUIC (quinn) даёт независимые стримы без HoL и лучшую переносимость потерь,
датаграммный UDP-транспорт в духе WireGuard ещё легче. Спроектировать: QUIC как
выбираемый транспорт (TCP оставить для роутера, где завязаны TPROXY и UDP
relay), 0-RTT возобновление, интеграция с криптоядром (Noise поверх QUIC либо
QUIC-крипто напрямую), совместимость с камуфляжем. Крупный дизайн, фазируется.

Серьёзность 0 (работает), Ценность высокая. Зависит от XR-061, смежно с XR-063.
