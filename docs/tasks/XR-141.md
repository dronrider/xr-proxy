# XR-141: Реализация LLD-29: импорт по URL (скоуп share:import, exec-плагины с yt-dlp, джобы с прогрессом, SSRF-гейт и песочница, качество height, бутстрап конфига, действие импорта в приложении)

Дизайн и решения в [LLD-29](../lld/29-share-url-import.md) (утверждён в XR-052);
тут только что сделано и как проверить. Ветка `xr-141-share-url-import`, серия
коммитов с ID в subject (proto -> hub -> share -> core -> jni -> android -> docs).

Реализовано по слоям:

- `xr-proto`: константа `SCOPE_IMPORT = "share:import"`, формат токена не
  менялся (новое имя в scope-строке v2).
- `xr-hub`: `invite_shares` минтит `share:read share:write share:import` там,
  где раньше минтил write (write-привязка + writable-запись); остальные пути
  минта как были read-only.
- `xr-share`: блок `[import]` + `[[import.plugin]]` в конфиге с fail-fast
  валидацией (`{url}` только отдельным элементом argv, `import` только при
  `writable`); модуль `import.rs` (таблица джоб в памяти, одна активная,
  очередь 4, URL-гейт по схеме/резолву/приватным диапазонам, роутинг по
  суффиксам хоста, зажим высоты `min(height, max_height)`, процесс на джобу в
  своей группе, песочница systemd-run c `IPAddressDeny` на Linux, парс
  `xr-progress`, таймаут и кап суммарного размера, публикация через
  хеш+fsync+rename с посевом `HashCache`, подметание `.xr-import-*` при
  старте); роуты `POST /{id}/import`, `GET`/`DELETE /{id}/import/{job_id}` с
  порядком гейтов из LLD п. 2.6; резерв всего неймспейса `.xr-` в safepath и
  обходе манифеста (каталоги тоже); CLI `share --import` с бутстрапом
  референс-блока и проверкой yt-dlp/ffmpeg в PATH; харнесс `xr-share import`
  с поллингом прогресса.
- `xr-core`: `import_url`/`import_status`/`import_cancel` поверх
  `direct_then_relay`, проверка `share:import` в скоупе гранта до сети,
  именованные ошибки `no_plugin`/`job_lost` для текстов UI.
- `xr-android-jni`: `nativeImportUrl`/`nativeImportStatus`/`nativeImportCancel`
  тонкими обёртками.
- `xr-android`: действие «Импорт по URL» в папке шары с `share:import` в
  гранте, диалог ссылки с чипами качества 720p/1080p/Максимум (default 1080p),
  строка «Импорт: N%» с крестиком-отменой, опрос раз в 2 с пока экран открыт,
  обновление листинга по done.

## Проверка

### 1. Автотесты (логика, без сети и без yt-dlp: плагин это скрипт, который тест пишет сам)

```sh
cargo test --workspace --exclude xr-client         # на маке xr-client не собирается
cargo test -p xr-share --features relay            # relay-сборка агента
```

Ключевое, что должно быть зелёным:

- скоуп и минт: `xr-hub` `share_v2::tests::test_import_scope_minted_with_write`
  (write-привязка даёт `share:read share:write share:import`, read-привязка нет);
- конфиг: `xr-share` `config::tests::test_import_config_parse`,
  `cli::tests::test_import_bootstrap`;
- гейты и джобы: `server::tests::test_import_happy_path` (202 -> running с
  прогрессом -> done, файл в dest с хешем в манифесте, джоб-папка убрана),
  `test_import_gates`, `test_import_url_guard_and_routing`,
  `test_import_failures`, `test_import_timeout_and_total_cap`,
  `test_import_height`, `test_import_cancel`, `test_import_output_filter`;
- модуль импорта: `import::tests::test_url_guard`, `test_plugin_routing`,
  `effective_height_clamps_to_owner_cap`, `parse_progress_is_lenient`,
  `build_argv_substitutes_literally`, `sweep_removes_only_service_leftovers`;
- резерв неймспейса: `manifest::tests::test_manifest_skips_xr_namespace`,
  `safepath::tests::test_path_traversal_blocked`;
- потребитель: `xr-core` `sync::tests::test_import_roundtrip`,
  `test_import_error_mapping`, `import_refused_without_scope_before_network`.

### 2. Ручной end-to-end на стенде

Полный сценарий из 14 шагов в [LLD-29 п. 6](../lld/29-share-url-import.md)
(харнесс с ноутбука, телефон, качество, отмена, рестарт агента, relay-путь).
Короткий смоук:

```sh
# на агенте (yt-dlp и ffmpeg уже стоят):
xr-share share /srv/dropbox --writable --import   # впишет [import], скажет "импорт: разрешён"

# с ноутбука:
xr-share import --invite <T> --share dropbox "<ссылка на короткий ролик>"
# прогресс бежит, по done файл с заголовком ролика лежит в шаре, pull его видит

# с телефона: в папке writable-шары иконка импорта -> ссылка -> строка
# "Импорт: N%" -> файл в листинге
```

## Отличия от исходного дизайна

Существенных нет. Мелочи реализации: у плагина `max_height` в конфиге имеет
default 1080 (в LLD поле было без default); таймаут и кап размера в тестах
крутятся через `JobSpec` напрямую (минутная гранулярность `timeout_min` для
теста слишком крупная); высота в JNI кодируется `<= 0` как «не задана».
