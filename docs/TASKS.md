# xr-proxy: задачи (префикс XR)

Канбан-доска открытой работы и единственный источник правды по тому, что в работе.
Строка задачи держит только метаданные и короткий заголовок; содержимое (разбор,
что сделано, сценарий проверки) живёт в файле [tasks/XR-NNN.md](tasks/), имя по
чистому ID. Файл заводится, когда появляется содержимое: однострочному бэклогу он
не нужен, LLD-задачи ссылаются прямо на [docs/lld/](lld/). Метаданные (тип, P, R,
статус) живут только в доске и в файле не дублируются. Готовую задачу из доски
переносим в [docs/TASKS-archive.md](TASKS-archive.md), её файл тем же коммитом
переезжает в `tasks/archive/<год закрытия>/` (ссылка в архивной строке сразу на
новое место). ID сквозные по репозиторию и не переиспользуются; ID ставим в
subject коммита там, где уместно (`feat(core): XR-9 ...`). Чувствительное (IP,
доступы, инфра) ни в доску, ни в файлы задач не пишем, оно остаётся в
гитигнорнутом `local-docs/`.

Приоритет `P` не ставим на глаз, он выводится из ранга `R` по системе `RANKING.md`
(в репозитории devkit рядом с проектом). Формула
`R = Серьёзность(0-75) + Ценность(0-10) + Неопределённость(0-5) + Баг(+5) + Рычаг(0-5)`,
колонка `R` держит сумму и разбивку в этом же порядке слагаемых. Бакеты: P0 при
R >= 75, P1 при 50-74, P2 при 25-49, P3 при 0-24 (смысл: P0 чинить немедленно, P1
мешает ежедневному использованию, P2 заметно улучшает, P3 терпит). Backlog
отсортирован по R убыванию, при равенстве по возрастанию ID. Типы: `bug` / `task` / `LLD`.
